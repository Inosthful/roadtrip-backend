## 📊 Schéma de base de données défini

Le schéma complet a été expliqué en détail et validé avec l'étudiant.

### Tables à créer (par ordre de dépendance)

#### 1. ✅ users (Déjà créée)
```
users
├─ id (PK)
├─ full_name
├─ email (UNIQUE)
├─ password (hashé)
├─ created_at
└─ updated_at
```

#### 2. ✅ auth_access_tokens (Déjà créée)
```
auth_access_tokens
├─ id (PK)
├─ tokenable_id (FK → users.id)
├─ hash
├─ expires_at
└─ ...
```

#### 3. ⏳ trips (À créer en priorité)
```
trips
├─ id (PK)
├─ title
├─ description
├─ start_date
├─ end_date
├─ budget (decimal)
├─ status (enum: 'planning', 'active', 'completed', 'cancelled')
├─ creator_id (FK → users.id)
├─ created_at
└─ updated_at
```

#### 4. ⏳ trip_participants (Table pivot Many-to-Many)
```
trip_participants
├─ id (PK)
├─ trip_id (FK → trips.id)
├─ user_id (FK → users.id)
├─ role (enum: 'creator', 'admin', 'member')
├─ invitation_status (enum: 'pending', 'accepted', 'declined')
├─ invited_at
├─ joined_at
├─ created_at
└─ updated_at
└─ UNIQUE(trip_id, user_id)
```

#### 5. ⏳ stops (Étapes du voyage)
```
stops
├─ id (PK)
├─ trip_id (FK → trips.id)
├─ title
├─ description
├─ type (enum: 'accommodation', 'restaurant', 'activity', 'poi')
├─ latitude (decimal 10,8)
├─ longitude (decimal 11,8)
├─ address
├─ google_place_id (nullable)
├─ order (integer - ordre dans l'itinéraire)
├─ is_locked (boolean - ordre fixe pour l'algo)
├─ arrival_date (nullable)
├─ departure_date (nullable)
├─ created_by (FK → users.id)
├─ created_at
└─ updated_at
```

#### 6. ⏳ photos (Photos des étapes)
```
photos
├─ id (PK)
├─ stop_id (FK → stops.id)
├─ user_id (FK → users.id)
├─ file_path
├─ caption (nullable)
├─ latitude (nullable)
├─ longitude (nullable)
├─ taken_at (nullable)
├─ created_at
└─ updated_at
```

#### 7. ⏳ expenses (Dépenses du voyage)
```
expenses
├─ id (PK)
├─ trip_id (FK → trips.id)
├─ stop_id (FK → stops.id, nullable)
├─ title
├─ description (nullable)
├─ amount (decimal)
├─ category (enum: 'transport', 'accommodation', 'food', 'activity', 'other')
├─ paid_by (FK → users.id)
├─ expense_date
├─ created_at
└─ updated_at
```

#### 8. ⏳ expense_splits (Partage des dépenses)
```
expense_splits
├─ id (PK)
├─ expense_id (FK → expenses.id)
├─ user_id (FK → users.id)
├─ amount (decimal - part de cette personne)
├─ is_paid (boolean - a remboursé ou pas)
├─ created_at
└─ updated_at
```

### Concepts clés expliqués à l'étudiant

**1. trip_participants (Many-to-Many)**
- Un user peut participer à plusieurs trips
- Un trip peut avoir plusieurs participants
- Table pivot nécessaire pour cette relation
- Bonus : roles et invitation_status

**2. is_locked dans stops**
- Ordre FIXE pour l'algorithme d'optimisation
- L'algo peut réorganiser les stops non-locked
- Les stops locked gardent leur position

**3. expense_splits (Système de balance)**
- Calcul de "qui doit combien à qui"
- Principe créditeur/débiteur
- is_paid track les remboursements

## Fonctionnalités principales attendues

### Gestion des voyages
- Création de roadtrips avec dates, titre, description et budget prévisionnel
- Invitation de participants par email avec système de roles (creator/admin/member)
- Statuts de voyage : planning, active, completed, cancelled

### Gestion des étapes (stops)
- Ajout d'étapes avec géolocalisation (latitude/longitude)
- Types d'étapes : hébergement, restaurant, activité, point d'intérêt
- Ordre personnalisable dans l'itinéraire
- Dates d'arrivée/départ par étape
- Upload de photos géolocalisées par étape
- Intégration possible avec Google Places API

### Optimisation d'itinéraire
- Algorithme d'optimisation pour calculer le meilleur ordre des étapes
- Prise en compte des étapes obligatoires (ordre fixe)
- Calcul des distances et temps de trajet

### Gestion financière collaborative
- Enregistrement des dépenses par catégorie (transport, hébergement, nourriture, activités)
- Système de split équitable ou personnalisé entre participants
- Calcul automatique des remboursements (qui doit combien à qui)
- Suivi budget prévisionnel vs réel

### Collaboration temps réel
- WebSockets pour synchronisation en temps réel des modifications
- Notifications pour invitations, ajouts d'étapes, dépenses

### Exports et rapports
- Export de l'itinéraire en PDF
- Récapitulatif financier du voyage
- Journal de voyage avec photos et notes
